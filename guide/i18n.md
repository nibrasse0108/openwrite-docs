# Internationalization

OpenWrite ships with built-in English and French locales and a flexible system for overriding any label, tooltip, or dialog string.

## Built-in locales

Pass a locale shorthand string to the `locale` prop:

```tsx
// English (default)
<OpenWrite locale="en" onChange={setHtml} />

// French
<OpenWrite locale="fr" onChange={setHtml} />
```

When `locale` is omitted, the editor defaults to `"en"`.

## Named locale exports

Both built-in locale objects are exported so you can inspect or extend them:

```ts
import { localeEn, localeFr } from 'openwrite'

console.log(localeEn.toolbar.bold)  // "Bold"
console.log(localeFr.toolbar.bold)  // "Gras"
```

## Partial override

You can override only the strings you care about by passing a `Partial<EditorLocale>` object. Unspecified keys fall back to the active built-in locale:

```tsx
import { OpenWrite } from 'openwrite'

<OpenWrite
  locale={{
    toolbar: {
      bold: 'Fett',         // override just this label
      italic: 'Kursiv',
    },
    placeholder: 'Hier tippen…',
  }}
  onChange={setHtml}
/>
```

::: tip Merging strategy
OpenWrite performs a **shallow merge** at the top-level keys (`toolbar`, `dialogs`, `wordCount`, etc.) and a **deep merge** within each key's object. This means you only need to specify the strings you want to change.
:::

## Full custom locale

For a fully translated UI, provide a complete `EditorLocale` object. Start from one of the built-in locales and extend it:

```tsx
import { OpenWrite, localeEn, type EditorLocale } from 'openwrite'

const localeEs: EditorLocale = {
  ...localeEn,
  toolbar: {
    ...localeEn.toolbar,
    bold: 'Negrita',
    italic: 'Cursiva',
    underline: 'Subrayado',
    strikethrough: 'Tachado',
    heading: 'Encabezado',
    link: 'Enlace',
    image: 'Imagen',
    table: 'Tabla',
    undo: 'Deshacer',
    redo: 'Rehacer',
    // … add all remaining keys
  },
  dialogs: {
    ...localeEn.dialogs,
    insertLink: {
      title: 'Insertar enlace',
      urlLabel: 'URL',
      textLabel: 'Texto',
      confirm: 'Insertar',
      cancel: 'Cancelar',
    },
    // … other dialogs
  },
  wordCount: {
    words: 'palabras',
    characters: 'caracteres',
  },
}

<OpenWrite locale={localeEs} onChange={setHtml} />
```

## Locale shape overview

The `EditorLocale` type has these top-level sections:

```ts
interface EditorLocale {
  toolbar: Record<BuiltinToolbarKey, string>  // button labels / tooltips
  dialogs: {
    insertLink:    { title: string; urlLabel: string; textLabel: string; confirm: string; cancel: string }
    insertImage:   { title: string; urlLabel: string; altLabel: string; confirm: string; cancel: string }
    insertTable:   { title: string; rows: string; cols: string; confirm: string; cancel: string }
    findReplace:   { title: string; find: string; replace: string; replaceAll: string; next: string; previous: string }
    sourceView:    { title: string; confirm: string; cancel: string }
  }
  wordCount: {
    words: string
    characters: string
  }
  placeholder: string
}
```

See the [Types reference](/api/types) for the full `EditorLocale` shape.

## Dynamic locale switching

Because `locale` is a regular React prop, you can switch languages at runtime without remounting the editor:

```tsx
import { useState } from 'react'
import { OpenWrite } from 'openwrite'
import 'openwrite/styles'

type Lang = 'en' | 'fr'

export default function MultiLangEditor() {
  const [lang, setLang]   = useState<Lang>('en')
  const [html, setHtml]   = useState('')

  return (
    <div>
      <div style={{ marginBottom: '0.5rem' }}>
        <button onClick={() => setLang('en')} disabled={lang === 'en'}>EN</button>
        <button onClick={() => setLang('fr')} disabled={lang === 'fr'} style={{ marginLeft: '0.5rem' }}>FR</button>
      </div>

      <OpenWrite
        locale={lang}
        value={html}
        onChange={setHtml}
        minHeight={300}
      />
    </div>
  )
}
```

::: warning Content vs. UI language
The `locale` prop only affects the **editor UI** (toolbar tooltips, dialog labels, placeholder text). It has no effect on the content the user types, which can be in any language.
:::
